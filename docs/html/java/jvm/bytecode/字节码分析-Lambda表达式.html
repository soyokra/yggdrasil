字节码分析-Lambda表达式

源码

package java.bytecode;

import java.util.function.Function;
import java.util.function.Predicate;

/**
 * Lambda表达式示例 - 演示Java 8 Lambda表达式
 */
public class LambdaExample {

    // Lambda表达式示例
    public void lambdaExample() {
        Function<String, Integer> func = s -> s.length();
        int length = func.apply("Hello");
        System.out.println("Length: " + length);
    }

    // Lambda表达式 - 多行
    public void lambdaMultiLineExample() {
        Function<String, String> func = s -> {
            String upper = s.toUpperCase();
            return upper + "!";
        };
        String result = func.apply("hello");
        System.out.println(result);
    }

    // Lambda表达式 - Predicate
    public void lambdaPredicateExample() {
        Predicate<Integer> isEven = n -> n % 2 == 0;
        boolean result = isEven.test(10);
        System.out.println("Is 10 even? " + result);
    }
}

详细说明

invokedynamic指令

Java 8 Lambda表达式使用invokedynamic指令实现，这是Java 7引入的动态调用机制：

0: invokedynamic #7,  0  // InvokeDynamic #0:apply:()Ljava/util/function/Function;

Lambda方法

编译器会为每个Lambda表达式生成一个私有静态方法（synthetic方法）：

private static java.lang.Integer lambda$lambdaExample$0(java.lang.String);
  flags: ACC_PRIVATE, ACC_STATIC, ACC_SYNTHETIC

方法命名格式：lambda$方法名$序号

BootstrapMethods

BootstrapMethods属性存储引导方法信息，用于在运行时动态生成Lambda实现类：

LambdaMetafactory

LambdaMetafactory.metafactory是引导方法，在运行时：
1. 创建Lambda实现类
2. 将Lambda方法绑定到函数式接口
3. 返回CallSite对象

Lambda实现机制

  1. 编译时：编译器生成Lambda方法（如lambda$lambdaExample$0）
  2. 运行时：invokedynamic调用引导方法
  3. 引导方法：使用LambdaMetafactory动态生成实现类
  4. 调用：通过函数式接口调用Lambda方法

优势

  • 延迟绑定：Lambda实现类在运行时生成，而不是编译时
  • 性能优化：JVM可以优化Lambda调用
  • 代码简洁：不需要为每个Lambda创建匿名内部类
